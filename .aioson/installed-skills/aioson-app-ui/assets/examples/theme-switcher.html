2. Tailwind vs Design System (mesmo resultado)

Os dois lados produzem o mesmo botão visualmente, e ambos respeitam o tema/modo ativo. A diferença é semântica: DS encapsula estados; Tailwind é composicional.

Via Design System
<button class="ao-btn ao-btn--primary">
Via Tailwind utilities (simulação)
class="bg-primary text-primary-on h-[34px] px-4 rounded-sm"
Importante: a coluna Tailwind aqui é simulada com tokens DS porque este HTML é standalone. Em projeto real com Tailwind v4 importado, as classes bg-primary e p-4 serão geradas automaticamente a partir do preset theme.css.

5. Critérios de aceite — Fase 1

Esta página valida visualmente os critérios definidos no Implementation Plan.

Tokens primitive completos — cyan, violet, neutral, green, amber, red, blue, plasma — 10 steps cada onde aplica
4 conjuntos completos de semantic tokens — dark-brand, dark-work, light-brand, light-work
Toggle funcional via [data-theme] × [data-mode] — sem JS além do switcher
Glow visível no brand mode, neutralizado no work mode — comparação no card hero acima
Cores em light mode com contraste WCAG — cyan #0891B2 em light vs #22D3EE em dark
Reduced motion respeitado — testado via @media (prefers-reduced-motion)
Focus states acessíveis — outline 2px com offset nos botões (use Tab para testar)
